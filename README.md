# OpenALEX Collector v2

OpenALEX APIを活用した学術論文収集・可視化ツール（v2）。
単一HTMLファイルで動作し、サーバー不要でブラウザだけで利用できます。

> すべての論文は、誰かの肩である。

## 概要

**OpenALEX Collector v2** は、[OpenAlex API](https://openalex.org/) を使って学術論文を検索・収集し、件数推移・引用ランキング・共同研究ネットワークを可視化するWebツールです。

- **単一HTMLファイル** — `OpenALEX_Collector.html` をブラウザで開くだけ
- **サーバー不要** — 完全クライアントサイド動作
- **共有が簡単** — ファイル1つを送るだけで誰でも利用可能

## v1からの主な追加機能

| 機能 | 説明 |
|------|------|
| 著者AND検索 | 著者名のセミコロン区切り入力で複数著者の共著論文だけを抽出 |
| 単独検索対応 | キーワード／所属機関／著者のいずれか1つでも検索可能 |
| 要約あり / 英語のみフィルタ | NLP・トピックモデル前処理での品質確保（デフォルトON） |
| 引用ランキング 6種 | 被引用論文・著者・機関・ジャーナル Top-N、件数×平均被引用 散布図、h-index |
| 共同研究ネットワーク 5種 | ペアランキング、エゴネットワーク、共同研究の広さ、ペア時系列、ヒートマップ |
| グラフ別CSV出力 | 表示Top-N外も含む全件をCSV出力（Excelで再描画・追加分析） |
| HTMLタグ除去 | `<i>`, `<jats:p>` 等のJATS残骸を自動除去 |
| 言語メタデータ | `language`（OpenAlex判定）と `title_script`（タイトル文字種）の2カラム |

## 機能一覧

### 論文検索

| 機能 | 説明 |
|------|------|
| キーワード検索 | スペース区切り = AND / 引用符 = フレーズ一致 |
| 複数クエリOR検索 | 1行1クエリ、複数行で自動OR統合（重複除去） |
| 所属機関フィルタ | 機関名でAND絞り込み。セミコロン区切りで複数機関OR指定可 |
| 著者フィルタ | 著者名でAND絞り込み。セミコロン区切りで複数著者AND指定可 |
| 単独検索対応 | キーワード／所属機関／著者のいずれか1つでも検索可能 |
| 年範囲フィルタ | 開始年〜終了年で絞り込み |
| 年別取得モード | 出版年ごとに検索し、10,000件/クエリ制限を回避 |
| 論文種別フィルタ | Article, Review, Book Chapter, Preprint 等 |
| ソート順 | 関連度順 / 引用数順 / 新しい順 / 古い順 |
| 最小引用数フィルタ | 指定引用数以上の論文のみ取得 |
| オープンアクセスフィルタ | OA論文のみに絞り込み |
| 要約ありのみ | OpenAlex の `has_abstract:true` フィルタ（デフォルトON） |
| 英語論文のみ | `language:en` + タイトルCJK等の後フィルタ（デフォルトON） |
| キャンセル機能 | 検索中にAbortController経由で中断可能 |

### データ表示

- ソート可能なデータテーブル（タイトル / 年 / 著者 / 所属機関 / ジャーナル / 引用数 / DOI）
- ページネーション（50件/ページ）
- DOIリンク付き
- メトリクス表示（取得件数 / 要約あり件数 / 期間 / CAGR / トレンド）

### グラフ可視化

検索結果は3つのカードに分かれた合計15タブで多角的に可視化されます。

#### 件数グラフ

| グラフ | 説明 |
|--------|------|
| 件数推移（棒グラフ） | 年別論文件数 + CAGR・トレンド表示 |
| 件数推移（折れ線） | 全体推移 / 著者比較モード（最大5名選択） |
| 著者ランキング | 横棒グラフ、表示数指定可（デフォルト20名） |
| 機関ランキング | 横棒グラフ、表示数指定可（デフォルト20機関） |

#### 引用ランキング

| グラフ | 説明 |
|--------|------|
| 被引用論文 Top-N | 個別論文を被引用数で横棒ランキング |
| 被引用著者 Top-N | 著者ごとの合計被引用数（共著者全員にフルカウント） |
| 被引用機関 Top-N | 機関ごとの合計被引用数 |
| 被引用ジャーナル Top-N | venue別の合計被引用数 |
| 件数×平均被引用数 散布図 | バブルチャート（X=論文数、Y=平均被引用、サイズ=合計被引用）、著者/機関切替 |
| h-index Top-N | 本コレクション内のh-index、著者/機関切替 |

#### 共同研究ネットワーク

| グラフ | 説明 |
|--------|------|
| 共同研究ペア Top-N | 「機関A × 機関B」のペアを共著件数で横棒ランキング |
| エゴネットワーク | 1機関を選択し、その機関の共同研究先を横棒表示 |
| 共同研究の広さ | 各機関の異なる共同先機関数（ユニーク件数） |
| ペア時系列 | 上位ペアの年別共著件数推移を折れ線で重ね描き |
| 共同研究マトリクス | 上位N機関のN×Nヒートマップ（HTMLテーブル） |

### エクスポート

- **CSV** — BOM付きUTF-8（Excel日本語対応）。論文一覧および各グラフの裏データ
- **グラフ別CSV** — Top-N表示外も含む全件出力（ランキング系は全著者・全機関）
- **PNG** — Canvasベースのグラフを画像保存（ヒートマップを除く）
- **CSVアップロード** — 以前保存したCSVを読み込み、再検索不要で可視化

## 使い方

### 1. ファイルを開く

`OpenALEX_Collector.html` をブラウザ（Chrome / Edge / Firefox / Safari）で開きます。

### 2. 検索を実行

1. **検索キーワード**を入力（複数行でOR検索）。空欄でも所属機関・著者で検索可能
2. 必要に応じて**所属機関フィルタ**（例: `Meijo University; The University of Texas at Austin`）と**著者フィルタ**（例: `Akira Yoshino; John Goodenough`）を入力
3. **年範囲**と**取得上限**を設定
4. 詳細フィルタで論文種別・最小引用数・OAのみ・要約ありのみ・英語のみを設定
5. 「検索を実行」をクリック

### 3. 結果を確認・可視化

- メトリクス（取得件数 / 要約あり件数 / 期間 / CAGR / トレンド）を確認
- データテーブルでソート・閲覧
- 3つのグラフカード（件数 / 引用 / 共同研究）でタブを切り替えて可視化

### 4. エクスポート

- 各グラフの「CSVダウンロード」で裏データを保存
- 各グラフの「PNGダウンロード」で画像保存
- 「CSVダウンロード」（論文データセクション）で論文一覧を保存

## 検索構文

```
キーワード検索:    "lithium-ion battery"        → スペース＝AND
フレーズ一致:      "lithium ion battery"        → 引用符で完全一致
OR検索:           "lithium-ion battery"         → 各クエリを個別検索し
                  "lithium ion battery"           重複除去して統合
機関フィルタ:      Meijo University; MIT         → セミコロン＝OR
著者フィルタ:      Akira Yoshino;                → セミコロン＝AND
                  John Goodenough                 （両者共著の論文だけ）
```

## 技術仕様

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML / CSS / JavaScript（単一ファイル、約3,500行） |
| グラフ描画 | [Chart.js v4](https://www.chartjs.org/)（CDN）＋ HTMLテーブル（ヒートマップ） |
| API | [OpenAlex REST API](https://docs.openalex.org/) |
| ページネーション | Cursor-based（10,000件制限なし） |
| レート制限 | 150ms間隔（polite pool: 10 req/sec） |
| リトライ | 最大5回、429エラー時は指数バックオフ |
| 文字コード | UTF-8（CSV出力はBOM付き） |
| h-index | 本コレクション内の論文・引用数のみで計算 |

## 注意事項

- OpenAlex APIは認証不要ですが、大量取得時はレート制限にご注意ください
- 所属機関・著者フィルタは入力名をOpenAlex APIで自動解決するため、部分一致の場合は意図しない機関・著者にマッチすることがあります（同名異人など）。解決後の正規名はステータスバーに表示されるので確認をおすすめします
- 年別取得モードは年数 × ページネーション回数分のAPIコールが発生するため、広い年範囲での大量取得には時間がかかります
- h-indexはOpenAlex本来の全著作h-indexとは異なり、本コレクション内に含まれる論文・引用数のみから計算されます
- 共同研究マトリクス（ヒートマップ）はHTMLテーブルベースのためPNG出力非対応です（CSVは出力可能）
- OpenAlex の `language` フィールドは要約ベース判定のため、`language=en` でもタイトルが多言語の論文が含まれることがあります（CJK等の自動除外フィルタを併用可能）
